Blender Integration

Cycles Game Engine is built directly into Blender in a form of standard add-on that can be applied to any Blender scene. Features like skinning, particle systems and physics are supported. This allows to author complete 3D content and see the changes immediatelly in the game from one place. A huge Blender ecosystem can be accessed seamlessly.

Material System

A subset of regular node-based cycles materials is used. Internally, shader variants are pregenerated and based on material nodes and model data a suitable render path is selected. For performance reasons, not all material nodes are supported currently.

The render pipeline is completely customizable, allowing use of custom shaders. Forward and deferred pipelines come as defaults.

Node Logic

Blender nodes can be used to create game logic visually, making it possible to prototype quickly. Nodes are then compiled to native code at compile time, ensuring small performance hit.

For more complex tasks, a native code can be written in Haxe language. Based on desired target platforms, C++ or JavaScript code can be used.

Ready for the Future

The foundation of engine is built on open source technologies. At core is Haxe programming language and Kha, which allows to render graphics on multiple platforms(HTML5, iOS, Android, Windows, OSX, Linux,...) supporting multiple graphics APIs(D3D9/11/12, OpenGL, WebGL, Metal, soon Vulkan). Kha does even more - it provides all the low level functionality and cross-compiles GLSL shaders.

Finally, Cycles Game(also open-source!) connects all parts together and integrates into Blender.
